Security Scan Report - E:\wps

后端未发现 SQL 注入、XXE/反序列化、用户输入 SSRF 或 admin CSRF/CORS 控制缺失。主要风险集中在 WPS USER/APP 授权边界。

B-001 High - Shared USER assertion key does not prove caller may act as requested user

全局 USER 断言密钥可以签任意 userId，而 WPS user token 又按 userId 全业务系统复用。需要增加业务系统到用户的授权证明。

B-002 Medium/High - APP preview can create preview links for arbitrary file IDs

fileId 只做长度校验，服务使用全局 WPS APP credential 创建预览。需要文件归属或授权校验。

B-003 Medium - Preview URL and expiry are trusted and returned directly

当前只校验 URL 非空和 expiry 可解析。应校验 HTTPS、host allowlist、无 userinfo、expiry 不超过请求窗口；如需强制有效期，需要网关短链/代理。

B-004 Medium - OAuth callback does not verify WPS subject

OAuth code 换回 token 后直接缓存到 state 的平台 userId 下，需要核对 WPS 授权主体。

Checked And Not Found

  • SQL injection: mappers use #{...}; no ${...}.
  • XML/XXE/deserialization: no app parser/deserialization sinks found.
  • CSRF/CORS: admin cookie writes have CSRF and allowlisted origins.
  • Secrets/logging: prod secrets are env-backed; no sensitive logging pattern found.
